Client driver silently truncates strings that exceed 32KB    Can be seen with this JUnit test case that retrieves a VARCHAR value with 20000 characters. With the client driver  the string is truncated to 10900 characters (32700 bytes when encoded in UTF-8). public void testLongColumn() throws SQLException { PreparedStatement ps = prepareStatement( 'values cast(? as varchar(20000))'); char[] chars = new char[20000]; Arrays.fill(chars  '/u4e10'); String str = new String(chars); ps.setString(1  str); JDBC.assertSingleValueResultSet(ps.executeQuery()  str); }
