Queries in metadata.properties allow tablepattern for JDBC methods that do not allow patterns    The DatabaseMetaData methods getBestRowIdentifier  getColumnPrivileges  getIndexInfo  getVersionColumns  getPrimaryKeys  getImportedKeys  getExportedKeys do not allow table name patterns. However  the SQL in metadata.properties implement these with 'table like ?'  thus allowing patterns. The SQL for these methods should be modified to 'table=?'.
