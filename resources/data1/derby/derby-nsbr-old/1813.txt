Fix hard-coded JDBC 4.0 type constants    The JDBC 4.0 driver has hard-coded type constants in JDBC40Translation. Some of these were changed in build 98 of jdk1.6 and should be updated. The nightly tests on jdk1.6 are failing because of this  see for instance http://www.multinet.no/~solberg/public/Apache/DerbyJDK16/testlog/JDK16Jvm1.6SunOS-5.10_i86pc-i386/440135-jdbc40_diff.txt.
