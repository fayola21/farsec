Access a web page that delivers a page with http Refresh header   2. Right click page  view source  3. Wait for Refresh timer (if 0  executes immediately on view source)    What is the expected result?    View page source should only show the raw source of the page  not execute   http Refresh commands delivered during the original download of the page.    What happens instead?    When you view source of a page that has Refresh http header  this gets   executed in view source.      Please provide any additional information below. Attach a screenshot if  possible.    Example response headers to reproduce issue:    HTTP/1.1 200 OK  Date: Sat  11 Jul 2009 06:10:38 GMT  Refresh: 5;URL=<a href="http://www.google.com/">http://www.google.com/</a>  ...    With this Refresh header  you would get redirected from view source page to   www.google.com 5 seconds after opening the view source.    -------------------------------------  HTTP/1.1 200 OK  Date: Sat  11 Jul 2009 06:07:07 GMT  Refresh: 1; javascript:alert('dont show this');    With the above refresh header  javascript gets executed on view-source  see    demonstration here:    <a href="http://ha.ckers.org/blog/20090710/running-javascript-in-chrome-despite-">http://ha.ckers.org/blog/20090710/running-javascript-in-chrome-despite-</a>  view-source/   </pre>
