<title>Issue 34619 -   chromium -    chrome.extension.sendRequest should require first parameter (either tabId or null) - Project Hosting on Google Code</title> <pre> The sendRequest prototype is:    chrome.extension.sendRequest(string extensionId  any request  function   responseCallback)    where extensionId is an optional argument.    The current implementation supports calling this function in the following   fashions:  sendRequest(extensionID  request  callback)  sendRequest(request  callback)    The expected implementation should support calling this function in the   following fashions:  sendRequest(extensionID  request  callback)  sendRequest(null  request  callback)    The expected behavior is more consistent with other chrome.* APIs that have   optional first parameters like chrome.tabs.executeScript or   chrome.tabs.getSelected.    Also see issue <a href="http://code.google.com/p/chromium/issues/detail?id=33049">http://code.google.com/p/chromium/issues/detail?id=33049</a>   regarding optional first parameters. </pre>
