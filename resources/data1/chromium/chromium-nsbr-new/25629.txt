<title>Issue 25629 -   chromium -    Visited link hash uses only 4 bytes of salt - Project Hosting on Google Code</title> <pre> In the function VisitedLinkCommon::ComputeURLFingerprint()  only the first  4 bytes of the salt are used to compute the URL fingerprint. This problem  is present in Subversion <a href="/p/chromium/source/detail?r=29883">revision 29883</a>.    VisitedLinkCommon::Fingerprint VisitedLinkCommon::ComputeURLFingerprint(      const char* canonical_url       size_t url_len       const uint8 salt[LINK_SALT_LENGTH]) {    [...]    MD5Update(&amp;ctx  salt  sizeof(salt));    [...]  }    When operating on an array which is a parameter to a function  the sizeof  operator returns the size of the pointer  not the size of the array. (See  &lt;<a href="http://www.lysator.liu.se/c/c-faq/c-2.html#2-6&gt;.">http://www.lysator.liu.se/c/c-faq/c-2.html#2-6&gt;.</a>)    The above line should be changed to the following:    MD5Update(&amp;ctx  salt  LINK_SALT_LENGTH); </pre>
