<title>Issue 32631 -   chromium -    Should not dispatch onload event after changing hash on about:blank - Project Hosting on Google Code</title> <pre> The following webpage never finishes loading:  <a href="https://ci.model.highpointins.com/prc/PRC/index.jsp?PLF_Code=HIPT">https://ci.model.highpointins.com/prc/PRC/index.jsp?PLF_Code=HIPT</a>    It gets caught in an infinite loop where it keeps refetching the favicon. The   problem is the page has a race where if one subframe (tmr3) finishes   loading before another (tmr2)  then it will end up changing the 'hash' on   &quot;about:blank&quot;. In Chrome this results in a new dispatch of onload()  in   other browsers it does NOT.    Here is a simplified reduction:    &lt;script&gt;     function displayPrivacy() {      alert(trm2.location);      trm2.location.hash = 'Privacy';    }  &lt;/script&gt;  &lt;iframe name=trm2 id=trm2 onload=&quot;displayPrivacy()&quot;   src=&quot;about:blank&quot;&gt;&lt;/iframe&gt;   </pre>
