<title>Issue 35454 -   chromium -    Add doc for CSS message replacement (extensions i18n) - Project Hosting on Google Code</title> <pre> Support for message replacement for external CSS (within extension   package) was implemented in <a href="http://codereview.chromium.org/570007">http://codereview.chromium.org/570007</a>.    It replaces messages in the form __MSG_some_name__ with translated   content.    Developers can use reserved (and any other) messages to specify text   direction  alignment...:    body {    dir: __MSG_@@bidi_dir__;  }    div#header {    margin-bottom: 1.05em;    overflow: hidden;    padding-bottom: 1.5em;    padding-__MSG_@@bidi_start_edge__: 0;    padding-__MSG_@@bidi_end_edge__: 1.5em;    position: relative;  }    For LTR languages this becomes:    dir: ltr;  padding-left: 0;  padding-right: 1.5em;    For RTL languages this becomes:    dir: rtl;  padding-right: 0;  padding-left: 1.5em;    They can also define messages that would increase/decrease font size   given the language  or resize dialogs given the language. </pre>
