<title>Issue 35250 -   chromium -    Launching from ObjectDock pops up 'The application was unable to start correctly'  fix involves using -no-sandbox command in shortcut. - Project Hosting on Google Code</title> <pre> When launching Google Chrome from ObjectDock  it opens and pops up 'The   application was unable to start correctly (0x00000a5). Click OK to close the   application.'  Clicking Ok and the popup repeatedly pops up.     The fix for this requires using the -no-sandbox command on the shortcut  thus   disabling one of the major security functions of chrome. </pre>
