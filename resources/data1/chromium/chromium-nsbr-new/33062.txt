<title>Issue 33062 -   chromium -    XMLHttpRequest.send sends 'undefined' string when passed undefined value - Project Hosting on Google Code</title> <pre> Chrome Version       : 4.0.249.78 beta (36714)           Safari 4: OK    Firefox 3.x: OK           IE 7: didn't test           IE 8: OK        1. Given the following JavaScript:    var x = new XMLHttpRequest();   x.open('POST'  '/');   x.send(undefined);    Expected result is:    POST request with empty request body.    Instead Chrome sends &quot;undefined&quot; as the request body.      Interestingly the following JavaScript    var x = new XMLHttpRequest();   x.open('POST'  '/');   x.send(null);    results in empty request body. Passing null or undefined results in empty body   in FF  IE  Safari.    Combined with default &quot;text/xml&quot; request content type (<a title="Post or Delete xhr with no content sends &quot;Content-Type: application/xml&quot; header"  href="/p/chromium/issues/detail?id=2507">Issue #2507</a>) this   breaks popular framework (read Ruby on Rails :) ) which implements strict   request body parsing (i.e. trying to parse &quot;undefined&quot; as XML fails).   </pre>
