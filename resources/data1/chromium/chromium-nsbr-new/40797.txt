<title>Issue 40797 -   chromium -    Slow load in test_shell with auto-detection of proxy settings - Project Hosting on Google Code</title> <pre> Tested on Windows Vista and Windows 7 64bit with Visual Studio 2005 and  Chromium <a href="/p/chromium/source/detail?r=42941">revision 42941</a>.      1. Configure the system to auto-detect proxy settings.  Control Panel -&gt;  Internet Properties -&gt; Connections -&gt; LAN Settings -&gt; Check &quot;Automatically  detect settings&quot;.  2. Run test_shell and load <a href="http://www.google.com">http://www.google.com</a>  3. Notice a multi-second delay before each resource is loaded.        Resources should load without a significant delay. The problem is that  CallWinHttpGetProxyForUrl() (netproxyproxy_resolver_winhttp.cc) calls  WinHttpGetProxyForUrl() which can then block for multiple seconds while  attempting to auto-detect proxy settings.    <a href="http://msdn.microsoft.com/en-us/library/aa383157%28VS.85%29.aspx">http://msdn.microsoft.com/en-us/library/aa383157%28VS.85%29.aspx</a> </pre>
