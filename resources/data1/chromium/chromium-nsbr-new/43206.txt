<title>Issue 43206 -   chromium -    Add &quot;clipboard&quot; permission to enable reading clipboard contents - Project Hosting on Google Code</title> <pre> Right now  pages can always access the clipboard via execCommand('copy') or   execCommand('cut'). This changed in <a href="http://trac.webkit.org/changeset/58703">http://trac.webkit.org/changeset/58703</a>.   For backwards compatibility  all extensions are currently allowed to access   the clipboard (see <a title="IMP: Need to enable clipboard inside extensions by using javascriptcanaccessclipboard property" class=closed_ref href="/p/chromium/issues/detail?id=43147"> issue 43147 </a>)  but for M6  we should add an explicit   &quot;clipboard&quot; permission bit that extensions can start using.    The experimental clipboard APIs will not be covered by this change. </pre>
