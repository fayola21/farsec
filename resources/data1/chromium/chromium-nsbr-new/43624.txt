<title>Issue 43624 -   chromium -    Move sync_proto target to avoid net/ dep on chrome/ - Project Hosting on Google Code</title> <pre> The sync_proto target in chrome/browser/sync/protocol exists to generate   python code that is used by testserver.py under /net.  This introduces an   undesirable dependency that breaks the upstream Webkit chromium port  since   the Webkit chromium port doesn't want to pull in anything from chrome/ (   <a href="http://trac.webkit.org/wiki/Chromium">http://trac.webkit.org/wiki/Chromium</a> ).    Darin's put in place a workaround: <a href="http://codereview.chromium.org/1996010">http://codereview.chromium.org/1996010</a>    We should fix the issue by pushing the sync_proto target (and the protocol   definitions) out of chrome/. </pre>
