<title>Issue 41271 -   chromium -    Use &lt;link rel=favicon&gt; for app tab icons instead of manifest icons - Project Hosting on Google Code</title> <pre> In an effort to reduce proprietary API  and to be more consistent with the future introduction of   persistent background pages  we should have web developers specify icons for app tabs using   &lt;link rel=favicon&gt; instead of using the manifest.    Since app tab icons are larger than the standard favicon size  we will need to implement this   new proposal for specifying the size:    <a href="http://dev.w3.org/html5/spec/Overview.html#attr-link-sizes">http://dev.w3.org/html5/spec/Overview.html#attr-link-sizes</a> </pre>
