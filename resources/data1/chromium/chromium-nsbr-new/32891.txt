Open the attached testcase      PASS      The SQLTransaction.executeSql accepts 4 parameters  the 1st mandatory    the other 3 optional. The 4th argument is an error callback.  The specification <a href="http://www.w3.org/TR/webdatabase/">http://www.w3.org/TR/webdatabase/</a>  #sqltransactionerrorcallback clearly mentions that if there is no error  callback  or the error callback did not return false  then rollback the  entire transaction and call the transaction error callback. Instead  in  Chrome and Safari  if there is no error callback  then the transaction  fails  else it always succeeds.    As a bonus  the specification also says that the transaction success and  error callbacks are also optional <a href="http://www.w3.org/TR/webdatabase/#database">http://www.w3.org/TR/webdatabase/#database</a>  yet  if null is passed  Chrome complains that the argument is invalid.  null means optional.   </pre>
