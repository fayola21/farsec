Click &quot;Start&quot;      Gmail opens  preloaded with the correct destination  subject and body fields.      The system mailto: handler opens. For many Windows users it's an unused and slow Outlook. On Linux the   link silently fails.    Note: The page does a window.open(&quot;<a href="mailto:pdf@koolwire.com?body=Thanks...&quot;">mailto:pdf@koolwire.com?body=Thanks...&quot;</a>). As far as I know  there   is no way for a content script to intercept such a request. The URL in itself works perfectly on a   static link [4].    Other example: mailto: links created via Ajax  as in [5].      If I understand the issue correctly  there is no way for an extension to fully handle mailto: links    and this is the source of countless bug reports from - otherwise happy - extension users.  A couple of proposed solutions:    1. Allow extensions to specify &quot;mailto: handler pages&quot; in their manifest  like:         &quot;protocol_handlers&quot; : { &quot;mailto&quot; : &quot;myhandler.html&quot; }  When the user clicks a link with a registered URL scheme  &quot;myhandler.html?url=...&quot; would be opened.    Pros:  - allows options (e.g: to support Google Apps for domains)  - local implementations possible (faster  increases privacy)  - easily updated      2.
