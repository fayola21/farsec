<title>Issue 13768 -   chromium -    InputScope support on Windows (a better IME integration) - Project Hosting on Google Code</title> <pre> This is applicable to both our UI and webkit text input field.     Windows &quot;InputScope&quot; integration    InputScope is a mechanism to describe text category for each input area    developed by Microsoft for their products  especially for Tablet UI.  <a href="http://blogs.msdn.com/tsfaware/archive/2007/07/10/input-scopes.aspx">http://blogs.msdn.com/tsfaware/archive/2007/07/10/input-scopes.aspx</a>  <a href="http://msdn.microsoft.com/en-us/library/ms698133.aspx">http://msdn.microsoft.com/en-us/library/ms698133.aspx</a>    A Google Summer of Code project  which aimed to make Mozilla TSF-aware  had   a plan to support Input Scope as &quot;-moz-input-scope&quot; property.  <a href="https://wiki.mozilla.org/Community:SummerOfCode08:TSF">https://wiki.mozilla.org/Community:SummerOfCode08:TSF</a>      (note: it's not clear what their plan is for other platforms)    GeckoTIP can add Input Scope support for Mozilla. See &quot;input scope   guessing&quot;  &quot;recognition filter&quot; in this FAQ  :   <a href="http://geckotip.mozdev.org/faq.html">http://geckotip.mozdev.org/faq.html</a>     </pre>
