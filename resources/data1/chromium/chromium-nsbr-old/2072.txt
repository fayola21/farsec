<title>Issue 2072 -   chromium -    Button padding mess - Project Hosting on Google Code</title> <pre> Text in the search button on <a href="http://topkayaker.net/TopKayakerShop/">http://topkayaker.net/TopKayakerShop/</a> gets   clipped on the right.    For &lt;button&gt; and &lt;input type=button&gt;  if it has a fixed width  then   browsers do all sorts of different things with the padding on the button.    IE: If you set a fixed width it respects padding if you set it  otherwise   centers the button contents in the button.    FF: If you set a fixed width totally ignores your *horizontal* padding and   (mostly) centers your content.    Chrome: If you set a fixed width it respects padding if you set it    otherwise acts as if padding:0 3px (which is the default padding in FF if   you don't set a fixed width).    Not totally sure what the right thing to do is. I lean towards having the   default padding change to padding:0 if you set a fixed width.    Haven't tested  but I presume the same all applies to fixed-height buttons. </pre>
