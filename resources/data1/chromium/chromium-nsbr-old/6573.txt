<title>Issue 6573 -   chromium -    Fix UI components that are created by WS_EX_LAYOUTRTL | WS_EX_RTLREADING flag - Project Hosting on Google Code</title> <pre> WS_EX_LAYOUTRTL | WS_EX_RTLREADING renders a RTL alignment but LTR reading   order.  Please see the following blog for detail:  <a href="http://blogs.msdn.com/michkap/archive/2007/03/11/1857043.aspx">http://blogs.msdn.com/michkap/archive/2007/03/11/1857043.aspx</a>    We need to replace the above flag with WS_EX_LAYOUTRTL only.    There are several UI that are created by using WS_EX_LAYOUTRTL |   WS_EX_RTLREADING flag  including:  1. AutoCompleteEditView (omnibox  related bug: 6101  6106  and perhaps   6108)    2. MenuHostWindow   3. TextField (for example  text in bookmark dialog  related bug: 6082)   4. NativeControl  including   4.1 checkbox  combobox  native_button  radio_button  separator    tabbed_pane  tree view.  4.2 table view (which should be fixed by <a href="http://codereview.chromium.org/18076">http://codereview.chromium.org/18076</a>)    All the above UI components need to be fixed to create correct UI layout.   This bug is used for tracking purpose.     </pre>
