<title>Issue 6285 -   chromium -    Autocomplete takes precedence over ctrl+enter in url bar. - Project Hosting on Google Code</title> <pre> Chrome Version       : 1.0.154.36 (Official Build 6676)             Safari 3: OK      Firefox 3: OK           IE 7: OK    When entering a domain that matches the beginning of another domain in your   history pressing ctrl+enter autocompletes the domain to match the history   entry rather than going to the entered domain.    For example  assuming you have a history entry for &quot;bankofamerica.com&quot;.  1) Enter &quot;ba&quot; in the url bar.  2) Hit ctrl+enter.    Expected result:  Should navigate to &quot;<a href="http://www.ba.com/">http://www.ba.com/</a>&quot;    Actual result:  Navigates to &quot;<a href="http://www.bankofamerica.com">http://www.bankofamerica.com</a>&quot;      Happens for all domains starting with the same characters as a domain in   the autcomplete history.     </pre>
