<title>Issue 1565 -   chromium -    Don't make GoogleUpdate always-resident - Project Hosting on Google Code</title> <pre> Product Version      : 0.2.149.27 (1583)    The Chrome install process installs GoogleUpdate.exe and enables it to   launch on startup.  The process sits in the background  running all the   time.    This is a disservice to the rest of the system.    Windows already has a Scheduled Tasks feature.  Use it.  Create a   scheduled task that calls GoogleUpdate.exe at regular intervals.    GoogleUpdate.exe can do its business and exit.    <a href="http://blogs.msdn.com/larryosterman/archive/2007/08/13/it-s-my-computer-">http://blogs.msdn.com/larryosterman/archive/2007/08/13/it-s-my-computer-</a>  dagnabbit-not-yours.aspx    <a href="http://blogs.msdn.com/larryosterman/archive/2007/08/15/so-why-are-applets-">http://blogs.msdn.com/larryosterman/archive/2007/08/15/so-why-are-applets-</a>  so-bad-anyway.aspx   </pre>
