<title>Issue 12359 -   chromium -    Chromium reports &quot;Invalid Server Certificate&quot; for self-signed server certificates - Project Hosting on Google Code</title> <pre> NSS reports the SEC_ERROR_CA_CERT_INVALID error (-8156) on a self-signed   server certificate.    Right now the NetErrorFromNSPRError function in ssl_client_socket_nss.cc maps   this error to ERR_CERT_INVALID in the 'default' case of the switch statement. ERR_CERT_INVALID is a fatal certificate error  so its SSL error page doesn't   have a &quot;Proceed anyway&quot; button.    SEC_ERROR_CA_CERT_INVALID should be mapped to ERR_CERT_AUTHORITY_INVALID   instead. </pre>
