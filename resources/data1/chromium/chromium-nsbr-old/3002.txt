<title>Issue 3002 -   chromium -    Allow certificate revocation checking failures - Project Hosting on Google Code</title> <pre> The new HTTP stack needs to allow certificate revocation checking  failures  i.e.  failures to download a CRL or connect to an OCSP  responder.    Although this has a slight security risk  this is the current  practice among browsers  and is allowed by the W3C Web Security  Context UI Guidelines  working draft 24 July 2008  Section 8.2  Certificate Status Checking Failures. </pre>
