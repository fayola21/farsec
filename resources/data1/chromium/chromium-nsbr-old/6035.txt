<title>Issue 6035 -   chromium -    Chrome incorrectly encodes % to &amp;amp;percnt; in width  height etc. attributes when saving pages - Project Hosting on Google Code</title> <pre> Chrome Version       : latest as of Jan 5th 2008              Firefox 3: OK           IE 7: OK      1. Copy the HTML code below to a text file table.html  2. Open table.html in Chrome  3. Save currently viewed file as table_saved.html  4. Open table_saved.html in Firefox 3      The table size should be 100% of window width when viewing table_saved.html  in Firefox 3.       The table size is rendered as 100 pixel wide by Firefox as Chrome has  encoded the table width of 100% into 100&amp;percnt; when saving the page.    Some more details:    When saving a web page with Chrome  each occurrence of a percent sign(%) is  encoded as &amp;percnt;. This leads to problems in attribute values like table  width  where &lt;table width=&quot;100%&quot; ...&gt; becomes &lt;table width=&quot;100&amp;percnt;&quot;  ...&gt; when saving the page with Chrome.    While Chrome still renders the saved page correctly  Firefox 3 interprets  the saved table width &quot;100&amp;percnt;&quot; as &quot;100&quot; pixels and renders the table  too small.    I don't think that Chrome should encode the percent sign (%) as the HTML  entity &amp;percnt; in width  length  height and other HTML entity attribute  size values when saving a web page.
