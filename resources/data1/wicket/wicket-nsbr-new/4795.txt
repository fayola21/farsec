Wicket unit tests rely on hashmap iteration order    Starting from version 7u5 java has supported randomizing the hashmap String key order (enabling this will fix CVE-2012-2739). It is also a good way to test the code if it is relying on hashmap order.More details: http://docs.oracle.com/javase/7/docs/technotes/guides/collections/changes7.htmlIn the latest Java 8 (b58) the randomized hashmap is enabled by default causing some wicket unit tests to fail randomly.
