UrlRenderer produces wrong full urls when the passed parameter is not absolute (as Url understands 'absolute')    Code like:Url baseUrl = Url.parse('one/two/three');baseUrl.setProtocol('http');baseUrl.setHost('www.example.com');baseUrl.setPort(8888);UrlRenderer renderer = new UrlRenderer(new MockWebRequest(baseUrl));renderer.setBaseUrl(baseUrl); // this is needed because MockWebRequest cuts dataString fullUrl = renderer.renderFullUrl(Url.parse('../four'));Produces url: http://www.example.com/../four while it has to be http://www.example.com/one/four
