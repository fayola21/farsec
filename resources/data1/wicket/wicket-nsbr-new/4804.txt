Add #setStatus(int) in AbstractResource.ResourceResponse    org.apache.wicket.request.resource.AbstractResource.ResourceResponse provides #setError(int code  String message) but doesn't provide #setStatus(int code).This makes it hard to set custom headers (e.g. cache control) and set status 304 because if HttpServletResponse#setErrorCode() is used then many headers are reset (including the cache control ones).
