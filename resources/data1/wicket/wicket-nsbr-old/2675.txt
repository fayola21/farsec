CryptedUrlWebRequestCodingStrategy fails in form with method='get'    A form fails to submit when it has method='get' set in a wicket application which is using CryptedUrlWebRequestCodingStrategy.This happens because the form is constructed by wicket like this:&lt;form wicket:id='form2' method='get' id='form26' action=''&gt; &lt;div style='display:none'&gt; &lt;input type='hidden' name='form26_hf_0' id='form26_hf_0' /&gt; &lt;input type='hidden' name='x' value='vcH4-yiA7z1xdg19IZiML85IPOzB2CaoynoU6UPAVS9n2G2S2bamuw' /&gt; &lt;/div&gt; &lt;input type='text' wicket:id='text2' value='' name='text2'/&gt; &lt;input type='submit'/&gt;&lt;/form&gt;which can produce a query string in firefox like this:http://localhost:8080/crypt-bug/?form26_hf_0=&amp;x=YhsA9niKQasfMz8FBHuRP1lLAZXPdZy8s1XWlmYcu7WwhnsP*VusYw&amp;text2=test'x' is the second parameter in the query string  but CryptedUrlWebRequestCodingStrategy accesses this parameter as '?x=' where in this case it is '&amp;x='I am attaching the example quickstart app and a possible fix.
