BigDecimalConverter is not precise    Currently  if we use the BigDecimalConverter class  it converts the values by first letting the NumberFormat to parse the value into a Double  which the BigDecimalConverter class in turn turns into a BigDecimal.Unfortunately such an implementation loses precision  for example if we enter '123.45' in a form  it can be converted to '123.45000000000001' or '123.44999999999999999' thanks to the conversion to a Double.Fixing the issue is rather simple: check if the NumberFormat you get is a DecimalFormat  and if yes  simply call format.setParseBigDecimal(true) and the NumberFormat will interpret the values correctly.
