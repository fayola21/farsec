Url to the home page is not correct when the current base url has only one segment    The link 'go back' in all Ajax examples in wicket-examples is not correct.In details: 1) current base url looks like : autocomplete?0 (i.e. one segment + one query parameter) 2) 'go back' points to the application home page (Index.class) 3) RequestCycle.mapUrlFor(IRequestHandler) creates Url('') for the home page 4) UrlRenderer.renderUrl(homePageUrl) incorrectly returns '' instead of '.' for this input
