CREATE TRIGGER doesn't detect all illegal references to SESSION schema    The reference manual's section on the DECLARE GLOBAL TEMPORARY TABLE statement says: Temporary tables cannot be referenced in a triggered-SQL-statementIt detects most cases and reports:ERROR XCL51: The requested function can not reference tables in SESSION schema.However  some references are not detected if they are in a subquery. For example:ij version 10.10ij&gt; connect 'jdbc:derby:memory:db;create=true';ij&gt; declare global temporary table temptable(x int) not logged;0 rows inserted/updated/deletedij&gt; create table t1(x int);0 rows inserted/updated/deletedij&gt; create table t2(b boolean);0 rows inserted/updated/deletedij&gt; create trigger tr1 after insert on t1 insert into t2 values exists(select * from session.temptable);0 rows inserted/updated/deletedThis CREATE TRIGGER statement should have raised an error.
