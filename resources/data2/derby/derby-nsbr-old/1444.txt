Using -Djvmflags option causes NoClassDefFoundError    When starting tests with the -Djvmflags option  the harness (RunList class) places '' around the argument to -Djvmflags when executing the java launcher to start a new jvm. This is (at least by suns java launcher) interpreted as a Class and not as a command line option  because of the ''. Because of this I get a NoClassDefFoundError.I believe this is a problem that happens only when using Suns java launcher.
