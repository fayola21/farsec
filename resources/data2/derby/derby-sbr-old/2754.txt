Client connections should not fail if permission is not granted to read derby.client.traceLevel  and derby.client.traceDirectory    Running the 10.2 network server tests with the 10.3 server  most tests fail with:AccessControlException: Access denied java.util.Property derby.client.traceLevel readdue to DERBY-1275. This can break existing applications that don't have this permission and should be a non-fatal warning if permission is not granted.
